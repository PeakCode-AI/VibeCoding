# 工单系统

工单系统是你与 VibeBase 运营团队沟通正式问题的渠道，适合需要跟进、有处理流程的事项。控制台「系统 → 我的工单」（`/app/console/my-tickets`）。

## 提交工单

1. 点击「提交工单」按钮，弹出对话框
2. 输入**标题**（简明描述问题）
3. 输入**描述**（详细说明现象、复现步骤、期望结果）
4. 点击「提交」

::: tip 写好工单
清晰的标题 + 完整的描述能显著缩短处理时间。建议在描述中包含：问题现象、发生时间、复现步骤、相关截图或订单号。
:::

## 工单列表

工单以表格形式展示：

| 列 | 说明 |
| --- | --- |
| 工单号 | 系统生成的唯一编号 |
| 标题 | 工单标题 |
| 状态 | 处理中 / 已解决 / 已关闭 |
| 优先级 | 高 / 中 / 低 |
| 创建时间 | 提交时间 |
| 查看详情 | 打开工单详情对话框 |

### 优先级

| 优先级 | 适用场景 |
| --- | --- |
| 高 | 账户无法登录、充值未到账、核心功能不可用 |
| 中 | 功能异常但有绕过方案、配置咨询 |
| 低 | 优化建议、一般性提问 |

## 工单详情

点击「查看详情」打开详情对话框，展示：

- 当前状态与状态变更时间
- 创建时间
- 完整问题描述
- 运营人员的处理回复（如有）

## 处理流程

工单提交后，会经过以下状态流转：

```
待处理（pending） → 处理中（processing） → 已解决（resolved） → 已关闭（closed）
```

1. 你提交工单，状态为 `pending`
2. VibeAdmin 运营人员接单，状态变为 `processing`
3. 问题处理完成，状态变为 `resolved`
4. 确认无误后归档，状态变为 `closed`

::: info 谁在处理工单
工单由 VibeBase 的 VibeAdmin 运营后台统一受理与分发。你可以在工单详情中看到处理进展与回复。
:::

## 工单还是反馈？

| 场景 | 推荐渠道 |
| --- | --- |
| 需要跟进、有 SLA 的正式问题（如充值、故障、开票） | 工单 |
| 对 AI 回复点赞 / 反对、产品建议 | 对话内反馈 |

::: tip 工单 vs 反馈
工单有正式的处理流程与状态跟踪，适合需要解决的问题；对话中的「喜欢 / 反对」按钮用于反馈 AI 质量，不进入工单系统。
:::
