# 角色权限

角色权限用于控制团队成员能访问哪些功能模块。通过为账号分配不同角色，可以实现精细化的权限管理。控制台「系统 → 角色权限」（`/app/console` 角色页面）。

## 角色概念

VibeBase 通过「角色 + 权限」的模型控制访问：

- **角色**：一组权限的集合，可分配给账号
- **权限**：对应具体功能模块的访问开关

常见的内置角色：

| 角色 | 说明 |
| --- | --- |
| 超级管理员 | 拥有全部权限，可管理一切 |
| 开发者 | 可管理 API Key、查看数据，通常不能改财务 |
| 只读成员 | 只能查看，不能修改 |

## 角色卡片

页面以**卡片网格**形式展示所有角色，每张卡片包含：

- 渐变色图标（视觉区分）
- 角色名称
- 成员数量
- 角色描述

点击卡片可查看该角色的权限配置详情。

## 权限矩阵

每个角色关联一组权限，权限按模块组织：

| 权限模块 | 控制范围 |
| --- | --- |
| 用户管理 | 子账号、角色配置 |
| 财务管理 | 充值、积分、消费记录 |
| API 管理 | API Key、能力配置 |
| 数据查看 | 用量分析、操作日志 |
| 系统设置 | 安全中心、公告 |

勾选某项即代表该角色拥有对应模块的访问权限。

## 新建角色

1. 点击「新建角色」按钮，弹出对话框
2. 填写以下信息：

| 字段 | 说明 |
| --- | --- |
| 角色名称 | 唯一标识，如「财务专员」 |
| 描述 | 角色职责说明 |
| 图标颜色 | 6 种预设颜色，用于卡片区分 |
| 权限 | 勾选该角色拥有的模块权限 |

3. 点击「确定」保存

## 编辑角色

在角色卡片上点击「编辑」，可修改名称、描述、图标颜色与权限。修改后立即生效，所有该角色的账号权限同步更新。

## 删除角色

点击「删除」并确认后，角色被移除。

::: warning 删除前检查
删除角色前请确认该角色下没有成员，否则相关账号会失去对应权限。建议先将成员迁移到其他角色。
:::

## 权限如何控制访问

当账号访问某个控制台页面或调用接口时，系统会：

1. 读取该账号的角色
2. 检查角色是否包含对应模块权限
3. 无权限则拒绝访问（页面隐藏 / 接口返回 403）

::: info 当前为演示数据
当前版本的角色与权限为模拟数据，用于展示权限模型与交互流程。生产环境的权限校验由后端 RBAC 实现。
:::
