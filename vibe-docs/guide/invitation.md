# 邀请系统

邀请系统让你通过专属邀请码邀请新用户注册。好友通过你的邀请注册后，双方都能获得积分奖励。控制台「系统 → 邀请系统」（`/app/console` 邀请页面）。

## 邀请码与链接

页面顶部展示你的专属邀请信息：

- **邀请码**：系统分配的唯一邀请码，点击「复制」可复制到剪贴板
- **邀请链接**：格式为 `{origin}/#/auth/sign-up?invite={code}`，点击「复制」可复制完整链接

将邀请链接发送给好友，对方打开即可进入注册页，邀请码会自动填入。

::: tip 一人一码
每位用户对应一个邀请码，长期有效。可重复使用，无需每次生成。
:::

## 分享邀请

点击「分享邀请」按钮，会调用浏览器的原生分享功能（`navigator.share`）：

- **移动端 / 支持原生分享的浏览器**：弹出系统分享面板，可选择微信、QQ、短信等
- **不支持原生分享的环境**：自动回退为复制邀请链接到剪贴板，并提示「已复制」

## 奖励规则

邀请奖励遵循以下规则：

| 规则 | 说明 |
| --- | --- |
| 奖励积分 | 邀请人与被邀请人各得 **2000 积分** |
| 邀请码 | 每人一个，长期有效 |
| 到账时间 | 被邀请人注册成功后**立即**到账 |
| 自我邀请 | 不允许（同一用户不能邀请自己） |

::: warning 不能自己邀请自己
系统会校验邀请人与被邀请人不能是同一账号。尝试自我邀请将无法获得奖励。
:::

## 邀请统计

页面中部以统计卡片展示你的邀请成果：

| 卡片 | 说明 |
| --- | --- |
| 总邀请 | 累计发出的邀请数 |
| 成功邀请 | 完成注册的邀请数 |
| 获得积分 | 通过邀请累计获得的积分 |

## 邀请记录

下方为邀请明细列表：

| 列 | 说明 |
| --- | --- |
| 被邀请人 | 被邀请用户的 ID |
| 邀请日期 | 发出邀请的时间 |
| 奖励积分 | 本次邀请获得的积分 |
| 状态 | 已完成 / 待处理 / 失败 |

### 状态含义

| 状态 | 说明 |
| --- | --- |
| 已完成（completed） | 被邀请人已注册，积分已发放 |
| 待处理（pending） | 邀请已发出，对方尚未注册 |
| 失败（failed） | 注册异常或触发风控，未发放奖励 |

::: info 注册需邀请码
新用户注册时必须填写邀请码。好友通过你的邀请链接打开注册页时，`?invite=` 参数会自动预填邀请码，无需手动输入。
:::
