# 用量分析

用量分析页帮助你掌握积分花在了哪里、哪些能力用得最多，从而优化成本。本页讲解如何阅读各项数据指标。

## 进入分析页

控制台「分析 → 用量分析」（`/app/console/analytics`），页面顶部为时间范围切换。

## 时间范围切换

支持三种时间区间，切换后页面所有数据同步刷新：

| 范围 | 说明 |
| --- | --- |
| 今日 | 当天 00:00 至当前 |
| 近 7 天 | 过去 7 个自然日 |
| 近 30 天 | 过去 30 个自然日 |

## 指标卡片

顶部 4 张卡片展示核心指标：

| 卡片 | 说明 |
| --- | --- |
| 总调用量 | 区间内接口调用总次数，附同比趋势 |
| 今日消耗 | 今日已消耗积分 |
| 本月消耗 | 本月已消耗积分 |
| 可用余额 | 当前可用积分 |

::: tip 关注趋势
「总调用量」卡片会显示与上一周期的同比变化（如 ↑12%）。如果消耗增速远超调用量增速，说明可能在高价模型上有浪费。
:::

## 调用趋势图

中部为**柱状图**，按天展示区间内的调用量变化：

- 鼠标悬停在某根柱上，会显示当天的**具体调用次数**与日期
- 可直观发现峰值日，进一步排查异常消耗

## 业务类型分布

以**进度条**形式展示不同业务类型的调用占比：

| 类型 | 说明 |
| --- | --- |
| AI 对话 | 文本对话调用 |
| 图像 | 图像生成 / 理解 |
| 语音 | 语音合成 / 识别 |
| 文本 | 文本处理 / 嵌入 |

进度条长度代表占比，可用于判断主要消耗方向。

## 能力调用排名

表格列出各能力（具体模型或接口）的调用情况，按调用次数倒序：

| 列 | 说明 |
| --- | --- |
| 排名 | Top 3 分别用金 / 银 / 铜徽章高亮 |
| 名称 | 能力名称 |
| 代码 | 能力标识码 |
| 调用次数 | 区间内总调用次数 |
| 百分比 | 占总调用的比例，以进度条展示 |

::: info 如何利用排名优化成本
排名靠前但单次成本高的能力，是最值得优化的对象。可考虑：
- 评估是否能切换到更便宜的同类模型
- 是否存在重复 / 无效调用
- 是否可以缓存高频结果
:::
