# 安全中心

安全中心用于保护你的账户安全，集中管理密码、两步验证、登录设备与操作日志。控制台「系统 → 安全中心」（`/app/console/security`）以四张卡片的形式组织。

## 修改密码

第一张卡片用于修改登录密码：

1. 输入**当前密码**
2. 输入**新密码**（最少 6 位）
3. 输入**确认密码**（需与新密码一致）
4. 点击「确认修改」

::: warning 密码强度
新密码长度不少于 6 个字符。建议使用字母、数字、符号组合，避免使用生日、手机号等弱口令。
:::

## 两步验证

第二张卡片用于开启 / 关闭两步验证（2FA）：

- 点击「启用」开启两步验证
- 点击「关闭」停用

::: info 本地存储说明
当前版本的两步验证状态保存在本地，用于演示开关流程。生产环境建议结合 TOTP（如 Google Authenticator）实现真正的二次校验。
:::

## 登录设备管理

第三张卡片列出所有登录过该账户的设备：

| 列 | 说明 |
| --- | --- |
| 设备 | 设备名称 / 浏览器 |
| 操作系统 | OS 信息 |
| 最后活跃 | 最近一次访问时间 |
| 当前设备 | 带有「当前」徽章标识 |

对非当前设备，可点击「下线」强制注销其登录态。

::: tip 发现异常登录
如果列表中出现你不认识的设备或异常 IP 来源地，请立即「下线」该设备并 [修改密码](#修改密码)，必要时开启 [两步验证](#两步验证)。
:::

## 操作日志

第四张卡片记录账户的关键操作，按时间倒序：

| 列 | 说明 |
| --- | --- |
| 操作 | 具体动作（如「登录」「修改密码」） |
| IP | 操作来源 IP |
| 浏览器 | UA 摘要 |
| 时间 | 发生时间 |
| 类型 | 成功 / 信息 / 警告，以不同颜色标识 |

- **成功**（success）：正常完成的操作
- **信息**（info）：中性记录（如查询）
- **警告**（warning）：需要留意的操作

## 安全建议

为保障账户安全，建议遵循以下原则：

- **定期更换密码**，至少每 3 个月一次
- **下线陌生设备**，发现可疑登录立即处理
- **开启两步验证**，为账户增加一道防线
- **不在公共设备上记住登录**，使用后及时登出
- **监控操作日志**，关注 `warning` 类型记录

::: danger 账户被盗怎么办
若确认账户被盗：① 立即修改密码 → ② 下线所有非当前设备 → ③ 检查 [操作日志](#操作日志) 是否有数据外泄 → ④ 必要时通过 [工单系统](./ticket) 联系运营冻结账户。
:::
