# API Key 管理

API Key 是你调用 VibeBase 后端接口的凭证。通过控制台的 API Key 管理页面，可以创建、查看、复制和删除 Key。本页讲解日常操作与安全注意事项。

## 进入 API Key 页

控制台「开发 → API Key」（`/app/console/api-key`），以表格形式展示当前账户下的全部 Key。

## Key 列表

| 列 | 说明 |
| --- | --- |
| 名称 | 你给 Key 起的备注名 |
| Key | 脱敏显示，格式 `vb-xxxx••••xxxx` |
| 状态 | 启用 / 禁用 |
| 创建时间 | Key 创建的时间 |

脱敏后只能看到前几位与后几位，中间用 `••••` 代替。

## 新建 Key

1. 点击右上角「新建 Key」按钮，弹出对话框
2. 输入 Key 名称（便于识别用途，如「测试服 Key」）
3. 点击「创建」

::: danger 完整 Key 仅显示一次
创建成功后，对话框会显示**完整的 Key 明文**，并提供「复制」按钮。这是你唯一一次看到完整 Key，关闭对话框后将无法再次查看。请立即复制并妥善保存。
:::

## 单个 Key 操作

每行 Key 右侧的操作区提供以下功能：

| 操作 | 说明 |
| --- | --- |
| 查看 | 查看完整 Key。系统会警告：完整 Key 仅在创建时展示一次，此后不再返回明文 |
| 复制 | 复制脱敏 Key（用于核对，非调用） |
| 删除 | 删除该 Key，需二次确认 |

::: warning 删除后立即失效
删除 Key 后，使用该 Key 的所有调用会立即返回 401 未授权。删除前请确认没有线上服务依赖此 Key。
:::

## Key 格式

VibeBase 的 API Key 统一以 `vb-` 前缀开头，后接一段随机字符串：

```
vb-<random>
```

调用接口时，将其放入请求头：

```
Authorization: Bearer vb-xxxxxxxxxxxxxxxxxxxx
```

## 安全最佳实践

- **妥善保存**：完整 Key 仅显示一次，建议存入密钥管理工具（如 Vault、密码管理器）
- **按需创建**：不同环境（开发 / 测试 / 生产）使用不同 Key，便于隔离与撤销
- **及时删除**：怀疑泄露时立即删除并重建 Key
- **不要提交到代码库**：避免将 Key 硬编码进代码或写入 `.env` 后提交
- **设置有意义的名称**：便于追溯每个 Key 的用途与归属

::: tip 多 Key 隔离
为不同子账号或不同项目创建独立 Key，配合 [子账号管理](./account) 可实现精细化权限与配额控制。
:::
