# 子账号管理

子账号是挂载在主账号下的成员单元，用于团队协作与配额控制。本页讲解如何创建、编辑和管理子账号。控制台「系统 → 子账号管理」（`/app/console` 账号管理页面）。

## 子账号是什么

子账号是主账号下的从属账号：

- **不独立登录**：子账号不能单独注册或登录，由主账号统一管理
- **共享主账号资源**：消耗主账号的积分池
- **独立配额**：可设置消费上限，防止单个成员超额
- **用于团队协作**：每个团队成员一个子账号，便于区分用量

## 搜索子账号

页面顶部提供搜索框，支持按以下字段查找：

- 账号 ID
- 用户名
- 昵称

输入关键词后实时过滤列表。

## 子账号列表

表格展示所有子账号：

| 列 | 说明 |
| --- | --- |
| 账号 ID | 系统生成的唯一标识 |
| 用户名 | 登录用户名 |
| 昵称 | 显示名称 |
| 消费限制 | 该子账号的积分消费上限 |
| 状态 | 正常 / 禁用 |
| 操作 | 编辑 / 禁用-启用 / 删除 |

## 添加子账号

1. 点击「添加子账号」按钮，弹出对话框
2. 填写以下信息：

| 字段 | 说明 |
| --- | --- |
| 用户名 | 子账号登录名 |
| 密码 | 初始密码 |
| 昵称 | 显示名称 |
| 消费限制 | 积分消费上限（`consume_limit`） |

3. 点击「确定」创建

## 编辑子账号

在操作区点击「编辑」，可修改昵称、消费限制等信息。

### 消费限制

`consume_limit` 控制该子账号在周期内可消耗的积分上限：

- 达到上限后，子账号的调用会被拦截
- 适合控制试用成员或外部协作者的用量
- 主账号可在 [积分消费记录](./points) 查看各子账号实际消耗

## 禁用与启用

点击「禁用」可临时停用子账号：

- 禁用后子账号无法使用任何功能（调用被拦截）
- 状态显示为「禁用」
- 可随时点击「启用」恢复

::: tip 禁用 vs 删除
禁用是临时措施，数据保留；删除是永久操作，不可恢复。不确定时优先禁用。
:::

## 删除子账号

点击「删除」并确认后，子账号及其数据被移除，且**不可恢复**。

::: danger 删除不可恢复
删除子账号会清除其消费记录与配置，删除前请确认无误。
:::
