# 适用场景

VibeBase 并非为单一场景设计，而是一个**可适应多种商业模式**的 AI 产品基座。本页列举典型的适用场景，帮助你判断它是否适合你。

## 场景一：独立开发者快速上线 AI 产品

::: tip 典型画像
你有技术能力，有 AI 应用的点子，但不想花两个月去搭用户系统、支付系统、运营后台。你希望最快速度把产品推到用户面前，验证能不能收费。
:::

**VibeBase 能帮你：**

- 跳过 80% 的「非核心但必须做」的基础设施工作（登录、积分、充值、订单）
- 直接专注于你的 AI 能力差异化（在 `abilities` 表新增定价、在 Service 层接入你的模型）
- 从源码到上线最快半天（Docker 一键部署）

**推荐组合：** VibeBase（Web）+ VibeBase API

**授权建议：** [学习版](https://github.com) 或 [专业版]

## 场景二：小团队 / 创业公司的 MVP 验证

::: tip 典型画像
2-5 人的小团队，正在做一个 AI 产品，需要快速验证商业模式。你需要多端覆盖（App + 小程序 + Web），需要运营后台看数据，但人手有限，无法每个端都从零开发。
:::

**VibeBase 能帮你：**

- **四端齐全** — Web、App、小程序、运营后台一次性到位
- **统一数据** — 无论用户从哪个端来，数据都进同一个库，运营后台统一管理
- **角色权限** — 团队成员按角色分工（开发 / 运营 / 只读）
- **可扩展** — 在成熟基座上叠加你的业务逻辑，而非重复造轮子

**推荐组合：** 全套四端 + VibeAdmin

**授权建议：** [专业版] 或 [企业版]

## 场景三：外包 / 交付团队的标准化交付物

::: tip 典型画像
你接了客户的 AI 项目，需要交付一套完整、可商用、可私有化部署的产品。客户要求源码、要求私有部署、要求有运营后台。
:::

**VibeBase 能帮你：**

- **完整交付** — 用户端 + 运营后台 + 全套文档，交付即生产可用
- **私有化部署** — 100% 可私有部署，数据库 / Redis / 对象存储全部客户自控
- **商用授权** — 企业版包含商用授权与私有部署许可
- **可定制** — 基于成熟代码做二次开发，比从零开始更可控、工期更可预期

**推荐组合：** 全套四端 + VibeAdmin + 私有化部署

**授权建议：** [企业版]（含商用授权）

## 场景四：企业内部 AI 能力平台

::: tip 典型画像
企业内部需要统一 AI 能力的入口，希望员工通过积分 / 配额的方式使用 AI，需要部门 / 角色管理，需要用量统计与成本管控。
:::

**VibeBase 能帮你：**

- **积分配额** — 子账号 + 消费限额，天然适合内部成本管控
- **用量分析** — 按能力 / 按时间的用量统计，成本透明
- **角色权限** — 按部门 / 职级分配 AI 能力访问权限
- **工单反馈** — 内部 IT 支持流程

**推荐组合：** VibeBase + VibeAdmin + 私有化部署

## 场景五：AI 能力 API 商业化

::: tip 典型画像
你有自研的 AI 模型或能力（图像生成、语音合成、OCR 等），希望以 API 形式对外开放并按调用量收费。
:::

**VibeBase 能帮你：**

- **AI 能力市场** — `abilities` 表定义能力，每项独立定价（按积分）
- **API Key 体系** — 用户自助申请 / 管理 Key
- **计费闭环** — 调用即扣分，积分不足即拦截，全程可审计
- **用量透明** — 用户和运营都能看到调用统计

**推荐组合：** VibeBase（API 为主）+ VibeAdmin（定价管理）

## 不适合的场景

::: warning VibeBase 可能不适合你，如果：
- **你只需要一个免费的聊天界面** — VibeBase 是商业化平台，不是个人玩具，有部署和配置成本
- **你需要深度定制 LLM 推理逻辑** — VibeBase 对接的是 OpenAI 兼容协议，自研推理引擎需自行适配
- **你的业务与 AI / 计费完全无关** — VibeBase 的核心价值在 AI 对话 + 计费闭环，纯 CRUD 业务用它略显重
- **你不愿意维护后端** — VibeBase 需要运行 PostgreSQL + Redis + FastAPI，如果你只想要静态站，它不适合
:::
