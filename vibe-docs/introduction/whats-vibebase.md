# 什么是 VibeBase

> 你要的不是又一套 Demo，是能收钱、能运营的 AI 产品。

VibeBase 是一套**面向最终用户的智能对话产品平台**，定位为「能收钱、能运营的 AI 商业化解决方案」。它不是聊天界面模板，而是一个包含完整商业闭环的产品工程：用户能对话、能充值、能续费；运营方能管人、管订单、管收入；四端（Web / 运营后台 / App / 小程序）齐全，共享同一套用户与账本。

## VibeBase 能做什么

::: info 一句话定位
**一次买断 · 源码交付 · 可私有化部署**的 AI 产品平台，从拿到源码到上线最快半天。
:::

VibeBase 内置了上线一个 AI 产品所需的全部能力：

- **AI 智能对话** — 流式 SSE 响应、多 Agent 编排、联网搜索、深度研究、深度思考、图像理解
- **完整的账户体系** — 注册 / 登录 / 邀请码 / 个人设置 / 安全中心 / 登录设备管理
- **积分与计费** — 积分账户、充值套餐、订单、支付回调、消费记录、用量分析
- **支付中台（VibePay）** — 免签约微信/支付宝收款，安卓监控端自动监听、异步回调入账，让「充值即到账」成为完整闭环（已部署 [pay.vibeadmin.cn](https://pay.vibeadmin.cn/)）
- **开发能力开放** — API Key 管理、AI 能力市场、开放接口，支持二次开发
- **运营管理** — 角色权限、子账号、工单、反馈、系统公告
- **四端覆盖 + 支付中台** — Vue3 用户 Web、运营后台、Flutter App、uni-app 小程序 / H5，以及独立的 VibePay 支付中台，五端齐全

## 和普通 AI 模板有什么不同

市面上的 AI 模板大多停留在「能对话」的阶段，给你一个聊天界面就结束了。VibeBase 解决的是**从对话到收钱到运营**的完整链路：

| 维度 | 普通 AI 模板 | VibeBase |
| --- | --- | --- |
| 对话能力 | ✅ 基础聊天 | ✅ 对话 + Agent + 搜索 + 研究 + 图像理解 |
| 用户体系 | ❌ 或仅 mock | ✅ 完整注册登录、邀请、双 Token 认证 |
| 计费闭环 | ❌ 无 | ✅ 积分 + 充值 + 订单 + 支付回调 + 消费记录 |
| 真实收款 | ❌ 需自己接商户 | ✅ VibePay 免签收款中台，付款即到账、回调即入账 |
| 运营管理 | ❌ 无 | ✅ 11 个控制台页面 + 角色权限 + 子账号 |
| 多端覆盖 | ❌ 单端 | ✅ Web / 后台 / App / 小程序四端 |
| 数据归属 | ❌ 托管在别人服务器 | ✅ 源码交付，数据在你手里 |
| 商业授权 | ❌ 不可商用 | ✅ 企业版支持商用与私有化部署 |

## 谁适合用 VibeBase

::: tip 三类典型用户
- **独立开发者** — 想快速上线一个能收费的 AI 产品，不想从零搭用户系统和支付
- **小团队 / 创业公司** — 需要多端覆盖和运营后台，希望第一天就能开始验证商业模式
- **外包 / 交付团队** — 需要一套完整、可商用、可私有化部署的交付物，减少重复造轮子
:::

## 技术栈一览

VibeBase 采用主流、稳定、易招聘的技术栈：

| 层级 | 技术 |
| --- | --- |
| 用户端前端 | Vue 3 + TypeScript + Vite + Tailwind CSS 4 + Pinia |
| UI 组件 | shadcn-vue（基于 Reka UI）+ lucide 图标 |
| 后端 | FastAPI + Uvicorn + Python 3.12 |
| 数据库 | PostgreSQL（与 VibeAdmin 共享） |
| 缓存 | Redis（限流 / Token 黑名单） |
| 认证 | JWT 双 Token（access + refresh） |
| AI 集成 | OpenAI SDK + LangChain Core（兼容通义千问） |
| 移动端 | Flutter（VibeApp） |
| 小程序 | uni-app（Vibe-Mp-H5） |

详见 [技术架构](./architecture)。
