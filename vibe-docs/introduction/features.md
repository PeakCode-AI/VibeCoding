# 核心特性

VibeBase 围绕「帮你赚钱、帮你省心、帮你做稳」三个维度，提供了 18 项核心特性。本页是全部能力的总览，每项特性都有对应的 [功能指南](../guide/dashboard) 和 [API 参考](../api/overview)。

## 帮你赚钱

商业化是 VibeBase 的第一性原理——不是能不能对话，而是能不能收钱。

### 1. AI 对话体验

完整的流式对话能力，是产品的核心入口。

- **流式 SSE 响应** — 基于 Server-Sent Events 的逐字输出，首字延迟低
- **双模式** — 普通对话（日常通用）与 Agent 模式（复杂任务自动化）
- **联网搜索** — 接入全网搜索，结构化展示来源、可点击查看详情
- **深度研究** — 多 Agent 编排（协调者 / 规划者 / 研究员 / 报告者），可确认计划、补充澄清、追踪步骤、下载报告
- **深度思考** — 可折叠的推理过程展示，支持结构化多 Agent 思考链
- **图像理解** — 支持上传图片进行视觉问答

详见 [功能指南 · AI 对话](../guide/chat) 与 [API 参考 · 对话](../api/chat)。

### 2. 充值与积分

完整的计费闭环，让 AI 能力真正变成收入。

- **积分账户** — 每用户独立积分账户，区分可用 / 已用 / 总量
- **充值套餐** — 基础 / 专业 / 企业多档套餐，支持微信 / 支付宝 / 银行转账
- **订单系统** — 完整的下单 → 支付 → 回调 → 到账流程，支持幂等
- **支付回调** — HMAC-SHA256 签名验证，对接真实支付网关
- **消费记录** — 按业务类型（对话 / 图像 / 语音）的消费明细，支持 CSV 导出

详见 [积分系统](../development/points-system) 与 [充值与支付](../development/recharge-payment)。

### 3. 多端获客

用户从哪个入口来都能接住，流量不流失。

- **四端覆盖** — Web、运营后台、Flutter App、小程序 / H5
- **统一登录** — 一套账号走遍四端
- **邀请裂变** — 邀请码系统，邀请人与被邀请人双向积分奖励

详见 [产品矩阵](./product-matrix) 与 [邀请系统](../guide/invitation)。

### 4. 开放能力

把 AI 能力变成可计费的商品，支持二次开发。

- **AI 能力市场** — 对话、文本生成、图像生成、语音合成、语音识别、OCR 等能力，各自独立定价
- **API Key 管理** — 用户可生成、启用、禁用、删除 API Key
- **开放接口** — 所有能力可通过 HTTP API 调用，适合集成到第三方系统

详见 [API 参考 · API Key](../api/apikey) 与 [API 参考 · AI 能力](../api/ability)。

## 帮你省心

运营一个产品，管理效率和数据透明同样重要。

### 5. 用量洞察

数据驱动的运营决策基础。

- **趋势图表** — 按小时 / 按天的调用量趋势
- **业务分布** — 对话 / 图像 / 语音 / 文本的占比
- **能力排名** — 各 AI 能力的调用次数排名与百分比
- **多时间窗** — 今日 / 近 7 天 / 近 30 天切换

详见 [功能指南 · 用量分析](../guide/analytics) 与 [API 参考 · 用量分析](../api/analytics)。

### 6. 控制台概览

登录后第一眼就能看到关键指标。

- 账户余额（含冻结金额）
- 今日消耗
- 最近交易（充值 + 消费合并展示）
- 一键跳转充值

详见 [功能指南 · 控制台概览](../guide/dashboard)。

### 7. 用户与权限

企业级的多角色管理。

- **角色系统** — 超级管理员 / 开发者 / 只读成员等角色自定义
- **权限矩阵** — 用户管理 / 财务管理 / API 管理 / 数据查看 / 系统设置五大权限
- **子账号** — 主账号下挂子账号，可设独立消费限额、可启用 / 禁用

详见 [功能指南 · 角色权限](../guide/role) 与 [功能指南 · 子账号管理](../guide/account)。

### 8. 经营看板

与 VibeAdmin 运营后台共享同一数据库，运营数据实时同步。

- VibeBase 的注册用户 = VibeAdmin 可运营的用户
- VibeBase 的充值订单 = VibeAdmin 收入统计的来源
- VibeBase 的工单 = VibeAdmin 工单处理模块的来源

详见 [多端协作 · VibeAdmin](../multi-end/vibeadmin)。

### 9. 工单与反馈

用户的声音能直达运营。

- **工单系统** — 用户提交、运营处理，状态流转（待处理 → 处理中 → 已解决 → 已关闭）
- **反馈中心** — 富文本反馈（基于 Tiptap 编辑器），支持联系方式留档
- **消息点赞 / 踩** — 对 AI 回复的点赞 / 反对，用于质量评估

详见 [功能指南 · 工单系统](../guide/ticket)。

### 10. 公告触达

版本更新、定价调整、维护通知的统一出口。

- 五种公告类型：系统维护 / 新功能 / 定价 / 版本 / 安全
- 置顶机制与时间线展示

详见 [功能指南 · 公告中心](../guide/announcement)。

## 帮你做稳

产品要长久，安全与稳定性是底线。

### 11. 安全与账号

企业级的安全防护。

- **JWT 双 Token** — access（7 天）+ refresh（30 天），支持无感刷新
- **Token 撤销** — 基于 Redis 黑名单，登出即失效
- **登录设备管理** — 查看 / 下线异常登录设备
- **操作日志** — 登录、改密等关键操作留痕
- **修改密码** — 支持旧密码验证，兼容 bcrypt 与 SHA-256 双格式

详见 [功能指南 · 安全中心](../guide/security) 与 [开发指南 · 认证机制](../development/authentication)。

### 12. 限流防护

防止滥用与攻击。

- **Redis 固定窗口限流** — 120 次 / 60 秒（按 IP + 路径）
- Redis 异常时自动放行，保证可用性

详见 [开发指南 · 认证机制](../development/authentication)。

### 13. 一致的体验

四端共享设计语言与交互模式。

- 统一的配色（品牌蓝 #3b82f6）、组件库、暗黑模式
- 统一的响应格式与错误处理

### 14. 清晰的消费

用户对自己的钱花在哪了一清二楚。

- 发送前预估 token 与积分消耗，余额不足直接拦截
- 实时 SSE 推送积分扣除事件
- 消费明细可按业务类型筛选、可 CSV 导出

### 15. 可控的交付

源码在你手里，数据在你手里。

- 一次买断，无订阅锁定
- 支持 100% 私有化部署
- 数据库、Redis、对象存储全部自主可控

### 16. 可预期的工期

从拿到源码到上线，时间可控。

- Docker 一键部署
- 环境隔离的配置体系（`config.{ENV}.yaml` + `.env`）
- 详细的部署文档（见 [部署](../deployment/docker)）

### 17. 团队可协作

前后端分离 + 清晰的分层，适合团队协作开发。

- 后端：Router → Service → DAO 三层架构
- 前端：apis → stores → components → views 分层
- 统一响应格式（`UnifiedResponseModel`）与统一错误处理

详见 [后端开发规范](../development/backend-conventions) 与 [前端开发规范](../development/frontend-conventions)。

### 18. 可扩展业务

在 VibeBase 基础上扩展新业务非常容易。

- 新增 AI 能力：往 `abilities` 表插一条记录即可定价上线
- 新增业务模块：参考现有的 Router/Service/DAO 分层快速复制
- 新增端：复用同一套 API，换一个前端壳
