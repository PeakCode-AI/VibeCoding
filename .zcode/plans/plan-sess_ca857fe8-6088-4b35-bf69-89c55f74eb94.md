# 先写计划文档,再执行生产部署

## 第一步:把部署计划写成文档存档
创建 `docs/guides/production-deploy.md`,把完整生产部署方案记录进去(目标、拓扑、端口、凭据处理原则、实施步骤、风险与回滚、安全提示)。这份文档作为后续执行的依据,也便于日后复盘/重新部署。

文档内容包含:
- 部署目标与访问方式(IP+端口,HTTP)
- 部署拓扑图(6+ 容器在同一 docker network,连接同机已存在的 PG)
- 端口对照表
- 凭据处理原则(`.env.production` 不入库,scp 上传)
- 实施步骤(探测环境→传代码→构建→启动→自动建表/初始化→验证)
- 数据初始化说明(VibeAdmin 自动建超管 admin@example.com/admin123;VibeBase 自动建表+6条 AI ability;不灌演示 seed)
- 安全提示(密码已暴露在对话,部署后建议更换 DB 密码)
- 风险与回滚、不做的事

同时创建配套生产编排文件(均不入库敏感部分):
- `deploy/prod/docker-compose.prod.yml`(编排,可入库)
- `deploy/prod/.env.production.example`(占位参考,可入库)
- `deploy/prod/.env.production`(真实密钥,**加入 .gitignore,不入库**)
- 在仓库根 `.gitignore` 追加 `deploy/prod/.env*`(保留 example)

## 第二步:探测服务器环境(只读)
SSH 到 118.31.224.254,确认:Docker/Compose 是否已装、能否连本机 `127.0.0.1:5432` 的 vibeadmin 库、相关端口占用情况、OS 架构。失败则停下汇报。
